응프화 화면구현 UI 개선 제안서 보기 자료

개선 전 화면과 개선 후 화면을 비교하며, 사용자가 겪는 불편함과 개선 효과를 근거 있게 정리하세요.

수행평가 과제 안내

제공된 웹페이지 화면과 학생 피드백을 바탕으로 UI 개선 제안서를 작성합니다. 답안은 단순 감상문이 아니라, 실제 사용자가 어떤 상황에서 왜 불편한지 설명하는 제안서 형식으로 작성합니다.

  • 제출 확인 위치를 못 찾음
  • 탭을 눌러도 바뀐 느낌이 약함
  • 모바일에서 표와 버튼이 불편함
  • 로그아웃 위치를 못 찾음

개선 전 화면

사용자가 원하는 기능을 찾기 쉬운지, 탭의 선택 상태가 분명한지, 모바일에서 표와 버튼을 사용하기 편한지 관찰하세요.

  • 제출 확인 탭이 눈에 잘 들어오는가?
  • 탭을 눌렀을 때 현재 위치가 분명한가?
  • 표 너비와 버튼 크기가 휴대폰에서도 적절한가?
  • 로그아웃 위치가 사용자가 예상하기 쉬운 곳에 있는가?

개선 후 화면

개선 후 화면에서 문제점이 어떻게 줄어들었는지 비교하세요. 단, 개선 후 화면도 완벽하다고 단정하지 말고 남은 개선 가능성도 생각해 볼 수 있습니다.

  • 선택된 탭이 색상과 굵기 등으로 더 분명해졌는가?
  • 현재 화면 제목이나 위치 안내가 더 명확해졌는가?
  • 버튼 크기와 위치가 터치하기 쉬워졌는가?
  • 로그아웃처럼 중요한 기능이 더 찾기 쉬워졌는가?

답안 작성 조건

1. 문제 사용자가 어떤 기능을 찾거나 조작할 때 불편한지 설명합니다.
2. 개선안 색상, 위치, 크기, 화면 배치, 반응형 처리 등 구체적인 수정 방향을 제시합니다.
3. 기대효과 개선 후 사용자가 무엇을 더 쉽게 할 수 있는지 설명합니다.
  1. 총 3개의 개선안을 작성합니다.
  2. 각 개선안은 문제 → 개선안 → 기대효과 흐름으로 씁니다.
  3. 각 개선안은 3~5문장 정도로 작성합니다.
  4. 최소 1개 이상은 모바일 사용성 개선을 포함합니다.

비교할 때 볼 기준

개선 전·후 관찰 기준
관찰 기준 확인할 내용 답안에 연결할 방향
정보 위치 제출 확인, 로그아웃 같은 주요 기능을 쉽게 찾을 수 있는가? 기능 위치와 사용자의 탐색 시간을 연결해 설명합니다.
상태 표시 탭을 눌렀을 때 현재 선택된 화면이 분명하게 보이는가? 색상, 굵기, 밑줄, 제목 표시 등 시각적 피드백을 연결합니다.
모바일 사용성 휴대폰에서 표를 읽고 버튼을 누르기 쉬운가? 표 배치, 가로 스크롤, 카드형 목록, 버튼 크기 등을 연결합니다.
