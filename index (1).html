개선 전 화면과 개선 후 화면을 비교하며 문제점과 개선 방향을 분석하세요.

보기 자료 안내

아래 두 자료는 같은 과제 관리 시스템 화면을 바탕으로 구성되었습니다. before.html은 사용성·가독성·모바일 대응 측면에서 문제점이 잘 드러나도록 일부러 불편하게 만든 예시이고, after.html은 그 문제를 어느 정도 개선한 예시입니다.

문제점 관찰

개선 전 화면

화면구현 관점에서 어떤 부분이 불편하거나 부족한지 실제로 눌러 보며 찾아보세요.

  • 탭 메뉴의 선택 상태가 잘 보이는가?
  • 모바일 화면에서 표를 확인하기 쉬운가?
  • 버튼 크기와 위치가 사용하기 편한가?
  • 로그아웃 위치를 쉽게 찾을 수 있는가?

개선 후 화면

개선된 화면에서 문제점이 어떻게 완화되었는지 비교해 보세요.

  • 활성 탭이 더 분명하게 표현되었는가?
  • 표 영역이 모바일에서도 확인 가능하게 처리되었는가?
  • 터치하기 쉬운 버튼 크기를 갖추었는가?
  • 중요 기능이 더 적절한 위치에 배치되었는가?

논술형 답안 작성 방향

단순히 “예쁘다 / 안 예쁘다”라고 쓰기보다, 사용자가 화면을 사용할 때 겪는 불편함을 근거로 설명하세요. 예를 들어 가독성, 접근성, 반응형 화면, 버튼 사용성, 정보 배치 같은 기준으로 비교하면 좋습니다.
